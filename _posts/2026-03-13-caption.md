![Caption](/assets/htb-linux/caption/caption.png)

**Caption**은 Hard 난이도의 Linux 머신으로, 기본 자격 증명을 통해 GitBucket에 접근하여 commit 기록에서 웹 포털 로그인 정보를 획득하고, **Web Cache Deception(`WCD`)**과 **XSS**를 이용한 응답 캐시 오염을 통해 관리자 세션을 탈취한다. 

이후 **HTTP/2 Cleartext 터널(`H2C Smuggling`)**을 사용해 HAProxy의 접근 제어를 우회하여 내부 서비스에 접근하고, 해당 서비스의 **Path Traversal 취약점(`CVE-2023-37474`)**을 악용해 **SSH ECDSA 개인 키**를 읽어 초기 foothold를 획득한다. 

마지막으로 루트 권한으로 실행되는 Apache Thrift 서비스의 **Command Injection 취약점**을 이용해 임의 명령을 실행하고 최종적으로 **루트 권한을 탈취**한다.

# Enumeration

## Portscan

먼저 대상 Host(`10.129.230.251`)에 대해 기본 스크립트와 서비스 버전 탐지를 수행하였다:

```bash
$ nmap -sC -sV 10.129.230.251 | tee nmap

Starting Nmap 7.95 ( https://nmap.org ) at 2026-03-12 08:34 EDT
Nmap scan report for caption.htb (10.129.230.251)
Host is up (0.26s latency).
Not shown: 997 closed tcp ports (reset)
PORT     STATE SERVICE    VERSION
22/tcp   open  ssh        OpenSSH 8.9p1 Ubuntu 3ubuntu0.10 (Ubuntu Linux; protocol 2.0)
| ssh-hostkey: 
|   256 3e:ea:45:4b:c5:d1:6d:6f:e2:d4:d1:3b:0a:3d:a9:4f (ECDSA)
|_  256 64:cc:75:de:4a:e6:a5:b4:73:eb:3f:1b:cf:b4:e3:94 (ED25519)
80/tcp   open  http-proxy HAProxy http proxy 2.0.0 or later
|_http-title: Caption Portal Login
|_http-server-header: Werkzeug/3.0.1 Python/3.10.12
|_http-open-proxy: Proxy might be redirecting requests
8080/tcp open  http       Jetty
|_http-title: GitBucket
Service Info: OS: Linux; Device: load balancer; CPE: cpe:/o:linux:linux_kernel

Service detection performed. Please report any incorrect results at https://nmap.org/submit/ .
Nmap done: 1 IP address (1 host up) scanned in 25.59 seconds
```

스캔 결과 `SSH(22)`, `HTTP(80, 8080)` 포트가 열려 있는 것을 확인하였다.

특히 `80` 포트는 **HAProxy**가 동작하고 있으며, 웹 페이지에 접근하면 **Caption Portal Login** 페이지가 나타난다.  
또한 `8080` 포트에서는 **GitBucket** 서비스가 실행되고 있는 것을 확인할 수 있다.

웹 페이지 접근 시 **caption.htb** 도메인으로 접근해야 함을 확인했다.
