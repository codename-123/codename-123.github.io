![Snoopy](/assets/htb-linux/snoopy/snoopy.png)

**Snoopy**는 Hard 난이도의 Linux 머신으로, **LFI** 취약점을 이용해 Bind9의 설정 파일에 포함된 `rndc.key` 을 탈취하는 것에서 시작한다. 

이후 해당 `rndc.key` 을 활용해 **mail 서브도메인을 공격자의 IP로 변경**하여 Mattermost의 비밀번호 재설정 메일을 가로채고 계정 접근 권한을 획득한다. 

서비스 내부에서는 웹 관리자가 원격 서버에 로그인할 때 사용하는 커스텀 플러그인을 조작해 공격자의 **SSH honeypot** 서버로 유도하고, 이를 통해 `cbrown` 의 자격 증명을 탈취한다. 

이어서 `cbrown` 의 권한을 활용해 `sbrown` 권한으로 `git apply` 를 실행할 수 있는 점을 악용하고, 심볼릭 링크 공격을 통해 권한 상승을 수행한다. 

마지막 단계에서는 **CVE-2023-20052** 취약점과 **XXE** 를 이용해 루트 사용자의 **SSH 키를 파일에 포함시키고, 해당 파일을 clamscan으로 검사하도록 유도**해 디버그 응답에 XXE 결과를 출력시키는 방식으로 최종적으로 루트 권한을 획득한다.
