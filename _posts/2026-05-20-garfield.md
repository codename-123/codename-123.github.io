![Garfield](/assets/htb-windows/garfield/garfield.png)

**Garfield**는 Hard 난이도의 Windows 머신으로, 초기에는 제공된 `j.arbuckle` 계정으로 도메인과 `SMB/SYSVOL` 을 열람하며 시작된다. 

이후 bloodyAD 로 쓰기 가능한 객체를 확인한 결과 `l.wilson` 및 `l.wilson_adm` 계정에 대한 제어 가능성을 발견하고, SYSVOL의 로그인 스크립트 경로에 악성 배치 파일을 업로드한 뒤 `l.wilson` 의 **scriptPath를 조작해 리버스 셸을 획득**한다. 

이후 `l.wilson` 권한으로 `l.wilson_adm` 의 비밀번호를 재설정하고 WinRM을 통해 관리자 계정으로 수평 이동한다. 

내부 정찰 중 RODC01이 별도 내부망에 존재하는 것을 확인하고, 이후 `l.wilson_adm` 이 RODC 관리 권한과 **SeMachineAccountPrivilege**를 가진 점을 이용해 `FAKE$` 컴퓨터 계정을 생성하고, RBCD를 설정하여 `Administrator` 로 가장한 CIFS 서비스 티켓을 발급받아 Ligolo로 터널을 구성하여 RODC01에서 `SYSTEM` 권한을 획득한다. 

RODC01에서 Mimikatz로 krbtgt_8245의 AES256 키와 도메인 SID를 추출한 뒤, Rubeus를 이용해 **RODC 기반 Golden Ticket 및 Key List 공격**을 수행한다. 

최종적으로 획득한 **Administrator 해시로 WinRM에 접속**하여 관리자 권한을 획득한다.

# Enumeration

## Portscan

먼저 대상 Host(`10.129.244.207`)에 대해 기본 스크립트와 서비스 버전 탐지를 수행하였다:

```bash
Th1sD4mnC4t!@1978
```

스캔 결과, Kerberos(`88`), LDAP(`389`, `636`, `3268`, `3269`), SMB(`445`), WinRM(`5985`), DNS(`53`), RPC(`135`, `593`), RDP(`3389`) 등의 서비스가 활성화되어 있었다.

또한 LDAP 배너 정보를 통해 도메인은 `garfield.htb`, 호스트는 `DC01.garfield.htb` 임을 확인하였다.

# Vulnerability Analysis

## SMB Enumeration

> 머신에서 주어진 사용자의 자격 증명은 다음과 같다: (`j.arbuckle` / `Th1sD4mnC4t!@1978`)

우선 SMB를 확인하였다:

기본적인 공유 디렉터리들이 확인되며, 이 중 `SYSVOL` 디렉터리를 살펴보았다:

```bash
$ smbclient //DC01.garfield.htb/SYSVOL -U j.arbuckle --password='Th1sD4mnC4t!@1978'
```

```text
smb: \garfield.htb\scripts\> ls

  .                                   D        0  Tue Jan 27 17:13:47 2026
  ..                                  D        0  Tue Jan 27 17:13:47 2026
  printerDetect.bat                   A      217  Fri Sep 12 18:20:29 2025

                9250815 blocks of size 4096. 987123 blocks available
```

내부에는 `printerDetect.bat` 파일이 존재하였다.

파일을 다운로드하여 내용을 확인하면 다음과 같다:

```bash
@echo off
echo Detecting installed printers...
echo ==============================

wmic printer get Name,DeviceID,PortName,DriverName,Shared,Status /format:table

echo.
echo Printer detection completed.
pause
```

해당 스크립트는 `wmic` 명령어를 이용하여 시스템 내 프린터 정보를 출력하는 단순 스크립트였다.

이후 SYSVOL 내부의 정책 파일을 추가적으로 확인하였다:

```text
smb: \garfield.htb\Policies\{6AC1786C-016F-11D2-945F-00C04fB984F9}\MACHINE\Microsoft\Windows NT\SecEdit\> get GptTmpl.inf

getting file \garfield.htb\Policies\{6AC1786C-016F-11D2-945F-00C04fB984F9}\MACHINE\Microsoft\Windows NT\SecEdit\GptTmpl.inf of size 3904 as GptTmpl.inf (4.8 KiloBytes/sec) (average 4.8 KiloBytes/sec)
```

GptTmpl.inf 파일 내부에는 다음과 같은 권한 설정이 존재하였다:

```text
��[Unicode]
Unicode=yes
[Registry Values]
MACHINE\System\CurrentControlSet\Services\NTDS\Parameters\LDAPServerIntegrity=4,1
MACHINE\System\CurrentControlSet\Services\Netlogon\Parameters\RequireSignOrSeal=4,1
MACHINE\System\CurrentControlSet\Services\LanManServer\Parameters\RequireSecuritySignature=4,1
MACHINE\System\CurrentControlSet\Services\LanManServer\Parameters\EnableSecuritySignature=4,1
[Version]
signature="$CHICAGO$"

[Privilege Rights]

# SKIP

SeBatchLogonRight = *S-1-5-32-559,l.wilson,*S-1-5-32-551,*S-1-5-32-544

# SKIP
```

여기서 `l.wilson` 사용자가 `SeBatchLogonRight` 권한을 가지고 있음을 확인할 수 있었다.

## scriptPath 

`SeBatchLogonRight` 는 사용자가 배치 작업 방식으로 로그온할 수 있도록 허용하는 권한이다.

즉, 해당 시스템에 직접 물리적으로 접근하지 않고도, 중앙에서 원격으로 혹은 작업 스케줄러를 통해 이 계정으로 프로그램을 실행할 수 있는 상태이다.

우선 현재 계정으로 도메인 객체에 대한 쓰기 권한을 확인하기 위해 [bloodyAD](https://github.com/CravateRouge/bloodyAD)를 사용하였다:

```bash
$ bloodyAD --host dc01.garfield.htb -d garfield.htb -u j.arbuckle -p 'Th1sD4mnC4t!@1978' get writable                                           

distinguishedName: CN=Guest,CN=Users,DC=garfield,DC=htb
permission: WRITE

distinguishedName: CN=S-1-5-11,CN=ForeignSecurityPrincipals,DC=garfield,DC=htb
permission: WRITE

distinguishedName: CN=krbtgt_8245,CN=Users,DC=garfield,DC=htb
permission: WRITE

distinguishedName: CN=Jon Arbuckle,CN=Users,DC=garfield,DC=htb
permission: WRITE

distinguishedName: CN=Liz Wilson,CN=Users,DC=garfield,DC=htb
permission: WRITE

distinguishedName: CN=Liz Wilson ADM,CN=Users,DC=garfield,DC=htb
permission: WRITE
```

결과를 통해 현재 `j.arbuckle` 계정이 `l.wilson` 사용자 객체에 대해 쓰기 권한을 가지고 있음을 확인할 수 있었다.
